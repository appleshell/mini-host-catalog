# 部署前检查清单

在部署到生产环境前，请确保完成以下所有项目：

## 📋 环境配置

- [ ] 已创建 `api/.env` 文件
- [ ] 已创建 `web/.env.local` 文件
- [ ] 已创建 `admin/.env.local` 文件
- [ ] API 的 CORS_ORIGIN 已配置为生产域名
- [ ] 管理员密码已设置为强密码（至少12位，包含大小写字母、数字、特殊字符）
- [ ] 所有 API_URL 已指向生产环境地址

## 🗄️ 数据库

- [ ] 数据库迁移已执行：`cd api && pnpm run db:migrate`
- [ ] 数据库文件权限正确
- [ ] 备份脚本已测试：`cd api && pnpm run backup`
- [ ] 已设置自动备份计划任务（crontab）
- [ ] 备份目录有足够磁盘空间

## 🔒 安全

- [ ] 所有 `.env` 文件已添加到 `.gitignore`
- [ ] 生产环境使用 HTTPS
- [ ] 防火墙已配置，只开放必要端口（80, 443）
- [ ] 管理后台使用强密码
- [ ] 已禁用不必要的调试信息

## 🚀 构建和部署

- [ ] 所有依赖已安装：`pnpm install`
- [ ] Web 前端已构建：`cd web && pnpm run build`
- [ ] Admin 前端已构建：`cd admin && pnpm run build`
- [ ] PM2 已安装：`pnpm add -g pm2`
- [ ] 服务已启动：`pm2 start ecosystem.config.js`
- [ ] PM2 已保存配置：`pm2 save && pm2 startup`

## 🌐 Web 服务器

- [ ] Nginx 已安装并配置
- [ ] 反向代理配置正确
- [ ] SSL 证书已配置（Let's Encrypt）
- [ ] 静态资源缓存已配置
- [ ] Gzip 压缩已启用

## 快速验证命令

```bash
# 检查服务状态
pm2 status

# 检查健康状态
curl https://api.yourdomain.com/health

# 查看日志
pm2 logs

# 测试备份
cd api && pnpm run backup
```

## 部署后立即做的事

1. 访问所有页面，确保正常工作
2. 提交一个测试线索
3. 登录管理后台查看
4. 检查日志是否正常写入
5. 验证备份是否自动执行

## 如果出现问题

1. 查看 PM2 日志：`pm2 logs`
2. 查看 Nginx 日志：`sudo tail -f /var/log/nginx/error.log`
3. 查看 API 错误日志：`tail -f api/logs/error-*.log`
4. 检查环境变量配置
5. 如需回滚，参考 DEPLOYMENT.md 中的回滚流程
